## 🏗️ 技术架构推断

### 前端技术
- **框架**：Next.js
- **UI 库**：Tailwind CSS
- **Web3 集成**：Privy（身份验证 + 钱包）

- **身份验证**：Privy API

- **代理网络**：分布式代理节点（5 个地区）
- **请求路由**：基于地区、负载、账号健康度

#### 1. 账号健康监控

#### 2. 请求路由

#### 3. 计量与计费
```

#### 4. 链上结算

## ⚠️ 风险点分析

### 1. 账号封禁风险
- **原因**：违反 Claude/OpenAI ToS（账号共享）
- **检测方式**：
  - 异常使用模式（24 小时不间断）
  - IP 地址频繁变化
  - 请求量远超正常用户
- **缓解措施**：
  - 代理 IP 轮换（5 个地区）
  - 请求速率限制
  - 模拟真实用户行为（间隔、时间分布）

### 2. 法律合规风险
- **ToS 违反**：明确禁止账号共享、转售
- **潜在后果**：
  - 批量封号
  - 法律函警告
  - 冻结未结算收益

### 3. 技术壁垒
- **核心能力**：
  - 分布式代理网络
  - 账号健康监控
  - 智能路由算法
  - 链上结算系统
- **复刻难度**：中等（需要时间积累）

## 🔑 核心 API 推断

### Provider 端 API

```
# 账户管理
GET  /api/provider/status      # 获取 Provider 状态
POST /api/provider/connect     # 连接 AI 账号
POST /api/provider/disconnect  # 断开 AI 账号
PUT  /api/provider/region      # 设置代理地区

# 收益查询
GET  /api/provider/earnings    # 收益统计
GET  /api/provider/requests    # 请求日志
GET  /api/provider/withdrawals # 提现记录

# 钱包管理

POST /api/provider/withdraw    # 手动提现（如有）
