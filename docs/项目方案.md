# DeRouter 复刻项目方案

## 📋 项目概述

**目标**：全套复刻 DeRouter.network - 基于智能合约的 AI 账号算力共享网络

**核心商业模式**：
- 用户托管闲置 AI 订阅账号（Claude/OpenAI/Codex 等）
- 平台将请求路由到托管账号
- 收益通过 Solana 链上自动结算
- Provider 获得被动收入，平台抽取 25% 佣金

## 🔧 技术栈分析

### 前端
- **框架**：React / Next.js（推测）
- **UI 库**：Tailwind CSS
- **状态管理**：Zustand / Redux
- **图表**：Recharts / Chart.js

### 后端
- **API 网关**：Node.js / Express 或 FastAPI
- **数据库**：PostgreSQL（用户数据）+ Redis（缓存）
- **链上结算**：Solana Web3.js
- **AI 路由**：自定义路由中间件

### 基础设施
- **托管**：Vercel / AWS
- **数据库**：Supabase / PlanetScale
- **智能合约**：Solana Program Library (SPL)
- **支付**：加密货币 + 微信支付

## 💰 商业模式拆解

### 收入来源
1. **API 调用差价**：Provider 成本 $20/月 → 平台收入 $40/月 → 毛利 $20
2. **平台抽成**：25% 佣金（Provider 75% / 平台 25%）
3. **充值沉淀资金**：用户预充值资金池

### 分销体系
| 模式 | 说明 | 适合人群 |
|------|------|----------|
| 推荐分销 | 分享推荐链接，获 30% 返佣 | 内容创作者、社区运营 |
| 客户密钥转售 | 创建密钥卖给客户，赚取差价 | API 分销商、SaaS 开发者 |
| 混合模式 | 推荐 + 转售双收益 | 综合运营商 |

## ⚠️ 风险提示

### 法律合规
1. **账号共享条款**：需确认 AI 平台 ToS 是否允许账号共享
2. **跨境支付**：加密货币结算可能涉及监管问题
3. **数据隐私**：用户 API 请求数据需加密处理

### 技术风险
1. **账号封禁**：AI 平台可能检测并封禁共享账号
2. **请求限流**：需实现智能限流避免触发风控
3. **链上成本**：Solana 交易费用波动
